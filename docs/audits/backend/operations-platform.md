# SaaS 运营与商城平台审计

## 核心决策

- **订阅管理**：`SubscriptionHandler` 管理租户订阅套餐，支持查询当前租户订阅和平台管理员更新订阅信息。按 `tenant_id` 过滤，每个租户只能看到自己的订阅。
- **平台横幅**：`BannerHandler` 实现 CRUD，管理平台轮播图。按 `sort_order` 排序，支持启用/禁用开关。平台级全局资源，不区分租户。
- **统计数据**：`StatsHandler` 提供平台仪表盘统计（机构数、资源数、GMV、订单数、提现待处理）和平台配置管理（手续费率、最低提现金额），以及机构自身收入统计。
- **机构管理**：`InstitutionHandler` 实现机构（学校/企业）完整生命周期：列表、创建、更新、审核、禁用。创建/更新使用事务管理标签。支持按状态/类型/搜索过滤，机构级数据隔离。
- **平台链接**：`PlatformLinkHandler` 管理外部平台链接配置（抖音、小红书等）。使用 `ON CONFLICT DO UPDATE` upsert 语义。平台级全局资源。
- **订单管理**：`OrderHandler` 实现教育资源交易订单完整流程：创建订单、模拟支付、查看订单详情、授权管理与验证。**`Pay` 方法使用事务包裹 5 个写操作（更新订单状态→更新卖家余额→更新买家支出→增加资源销量→创建授权记录）**，是典型的资金交易事务。
- **提现管理**：`WithdrawalHandler` 管理机构提现申请：列表、创建提现、审核提现状态。创建时校验最低金额和余额充足性。
- **应用模块**：`AppModuleHandler` 管理平台应用模块定义，按平台分组展示，支持 CRUD 操作。平台级全局资源。

## 检查点

| 检查点 | 结论 | 说明 |
|---|---|---|
| 订阅管理 | PASS | 按租户隔离；平台管理员可更新 |
| 平台横幅 | PASS | CRUD；排序；启用开关；平台级资源 |
| 统计数据 | PASS | 仪表盘聚合查询；平台配置管理；机构级隔离 |
| 机构管理 | PASS | 完整生命周期；事务管理标签；机构级隔离 |
| 平台链接 | PASS | CRUD；upsert 语义；平台级资源 |
| 订单管理 | PASS | 完整交易流程；资金事务；防重复购买；租户隔离 |
| 提现管理 | PASS | 最低金额校验；余额充足性检查；租户隔离 |
| 应用模块 | PASS | CRUD；按平台分组；平台级资源 |

## 风险与约束

- **提现创建已有事务包裹**：`Create` 方法在 `withdrawal_handler.go:133` 已使用 `Begin` + `Commit` 事务包裹 INSERT 和 UPDATE 两步，资金操作一致性有保障。—— **当前已覆盖，无需额外处理。**
- **订单 Pay 事务复杂度高**：单事务涉及 5 个写操作，并发场景下可能影响性能。—— **核心交易流程，当前可接受，后续关注。**
- **机构 Create 无显式鉴权**：`InstitutionHandler.Create` 没有显式鉴权检查，可能存在未授权创建机构的风险。—— **需确认是否为产品设计（开放注册）或遗漏鉴权。**
